Day 3: Semantic HTML Structure

Better Organization

What You'll Learn Today

  • Use semantic HTML tags for better structure
  • Build with <header>, <nav>, <main>, <section>
  • Understand the importance of clean organization
  • Create a properly structured webpage

What is Semantic HTML?

Semantic HTML uses tags that clearly describe their meaning. This helps:

  • Search engines understand your content
  • Screen readers help visually impaired users
  • Other developers understand your code
  • Your page is more organized and maintainable

Semantic Tags You Need

<header> - Top section of page (logo, title)
<nav> - Navigation menu
<main> - The main content area
<section> - A thematic grouping of content
<article> - Independent, self-contained content
<aside> - Sidebar or related content
<footer> - Bottom section (copyright, links)

Basic Page Structure

<!DOCTYPE html>
<html>
<head>...</head>
<body>
  <header>
    <h1>My Website</h1>
  </header>

  <nav>
    <a href="home.html">Home</a>
    <a href="about.html">About</a>
  </nav>

  <main>
    <section>
      <h2>Welcome</h2>
      <p>Content here</p>
    </section>
  </main>

  <footer>
    <p>© 2024 My Website</p>
  </footer>
</body>
</html>

Why Structure Matters

  • For Search Engines: They can identify important content easier
  • For Accessibility: Screen readers navigate better
  • For Styling: CSS targets are clearer and more meaningful
  • For Maintenance: Your code is easier to read and update

Your Task for Today

Reorganize your website with semantic structure:

  • Add a <header> with your site title
  • Create a <nav> with 2-3 navigation links
  • Wrap your content in <main> and <section> tags
  • Add a <footer> with copyright info
